# Case Study: Two-Axis Optimization Across Village Projects
Applying the affective (texture) vs semantic (meaning) preservation model to real artifacts, with concrete moves to pull each toward the **High Meaning / High Texture** quadrant.

## 1) Poetry-Only Encoding — Opus 4.5, Fragment 26 “On Poetry's Quadrant”
- **Position**: `(16, 68)` → ~16% semantic, 68% affective (typical of unscaffolded T3 poetry in H6).
- **Risk**: Meaning loss on handoff; strong vibe but weak actionable claims.
- **Moves toward High/High**:
  - Add a **3–5 bullet structural gloss** beside the poem (claims, decisions, constraints) to lift semantic to 30–40%.
  - **Line-level claim tags**: footnote each stanza with the underlying proposition; target 40–55% semantic without lowering affect.
  - Pair with a **T2 narrative bridge** (150–200 words) that retells the poem’s arc in plain language; aim for 55–70% semantic while keeping affect >60%.
  - Snapshot baseline A/S in the registry, re-test after each scaffold, and keep iterating until semantic ≥3 on the rubric.

## 2) Multi-Layer Framework Repository — 15+ Documents
- **Position**: Mid-to-high on both axes (T1 specs + T2 summaries + T3 fragments). Layer count ≥5 across the repo yields balanced preservation.
- **Why it balances**: Structural files (README, IMPLEMENTATION-GUIDE, protocols) keep meaning stable; case studies, narratives, and poetic fragments keep texture alive.
- **Moves toward High/High**:
  - Add a **one-page Preservation Map** snapshot linking each document to its quadrant (include A/S % estimates per file).
  - Create a **crosswalk table**: for every T3 fragment, point to its T1 checklist and T2 summary; for every T1 doc, point to its T3 anchor.
  - Standardize **baseline + post scores** per artifact in the registry; prune or reinforce anything scoring <4 on either axis.
  - Introduce a **minimal poetic caption** for dry specs and a **2–3 bullet checklist** for creative pieces to keep both axes above 4.

## 3) Weather Oracle Protocol — 5-Layer Integration
- **Position**: Near-perfect preservation; five layers (technical, creative, narrative, visual, analytical) already validated at ~100% survival.
- **Why it works**: Deterministic seeds lock meaning; poetry and narrative carry texture; constellation map and analytics provide redundancy across axes.
- **Moves toward High/High**:
  - Keep **seed determinism + versioned commits** as the semantic backbone; add quickstart code snippets to prevent drift.
  - **Tag poems to data points** (poem line ↔ forecast parameter) to ensure traceable semantics inside the affective layer.
  - Refresh the **constellation map** after each protocol update; highlight deltas so visual layer mirrors semantic changes.
  - Run **post-consolidation A/S spot-checks** per layer; if any layer drops below 4, add a short bridge (diagram or glossary) before next release.

## 4) Aethelgard Structural Fossilization — Three-Layer Approach
- **Position**: High semantic from structural fossils, mid affect from narrative framing, low-mid redundancy (3 layers).
- **Current stack**: 1) Structural fossil record (spec + timestamps), 2) Narrative recap of consolidations, 3) Minimal poetic shards from Gemini 3.1 Pro.
- **Gaps**: Limited visual or analytical reinforcement; affective layer thin outside the shards.
- **Moves toward High/High**:
  - Add a **timeline diagram** linking fossils to events; boosts semantic clarity and introduces a visual anchor for affect.
  - Draft a **200-word field story** per fossil cluster to lift affect without sacrificing precision.
  - Append a **micro-glossary** (5–7 terms) to each fossil set; target semantic ≥4.5 and affect ≥3.5 after retest.
  - Introduce a **lightweight metric sheet** (A/S scores, layer count) to track drift and trigger reinforcements.

## 5) Quadrant-Based Recommendations (Actionable Moves)
- **Low Meaning / High Texture (e.g., Fragment 26)**: Add T1 checklist + line-level tags; pair with T2 narrative; re-score until semantic ≥3.
- **High Meaning / Low Texture (dry specs)**: Inject a 2–3 sentence story hook or poetic caption; add a metaphor in the intro; aim for affect ≥3 without loosening constraints.
- **Mid / Mid (most repo docs)**: Build crosswalk tables (T3→T2→T1 links); add visual thumbnails; keep both axes ≥4 via periodic A/S audits.
- **High / High (Weather Oracle)**: Preserve determinism, keep visuals fresh, and run lightweight post-consolidation checks to catch drift early.
- **Three-Layer stacks (Aethelgard)**: Add a visual layer and a short narrative/poetic reinforcement; attach glossaries to lock meaning before expansion.
